Analysis

The Problem Definition

You are given three pegs and a stack of n disks, each disk having a different size.
All the disks are initially placed on one peg in decreasing size from top to bottom, i.e., the largest disk is at the bottom
and the smallest is at the top. The goal is to move the entire stack to another peg, following these rules:

1. Only one disk can be moved at a time.

2. A disk can only be moved if it is the top disk on a peg.

3. A disk can only be placed on an empty peg or on top of a larger disk.

Diagram

Space Complexity

The space complexity is O(2^n) because the number of moves increase exponentially as the number of disks increase.

Recursive Solution

Base Case :

If there is only one disk, simply move it from the source peg to the destination peg.

Recursive Case :

1. Move n-1 disks from the source peg to the auxiliary peg (using the destination peg as an intermediary).
2. Move the largest disk (the nth disk) directly from the source peg to the destination peg.
3. Move the nâˆ’1 disks from the auxiliary peg to the destination peg (using the source peg as an intermediary).

Number of Moves :

The number of moves required to solve the Tower of Hanoi problem for n disks is T(n), where T(n)= 2^n -1.

Example for small n :

For n = 3, the minimum number of moves are T(3) = 2^3 -1 = 7.

The steps would be :

1. Move disk 1 from peg A to peg C.

2. Move disk 2 from peg A to peg B.

3. Move disk 1 from peg C to peg B.

4. Move disk 3 from peg A to peg C.

5. Move disk 1 from peg B to peg A.

6. Move disk 2 from peg B to peg C.

7. Move disk 1 from peg A to peg C.
